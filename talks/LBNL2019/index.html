FlowPM : Differentiable fast N-body simulations in TensorFlow

the fast Particle-Mesh Simulations

The idea : move particles with approximate gravitational forces by estimating densities on grid

The numerical scheme
  • Interpolate particles on the grid to estimate mass density

  • Estimate gravitational force on grid points with FFT

  • Interpolate the force at particle positions

  • Kick : Update the momentum of the particles

  • Drift : Update the position of the particles

    • $\longrightarrow$ Repeat!
Cloud-in-cell (CIC) binning $$ W(r) = \begin{cases} 1 - \frac{r}{d} & r\leq d \\ 0 & otherwise \end{cases} $$

$$\nabla^2 \phi = - 4\pi G \rho$$ $$f(\vec{k}) = i\vec{k}k^{-2} \rho(\vec{k})$$

$$p(t_1) = p(t_0) + f(t_r) \mathcal{K}_\mathrm{PM}$$

$$x(t_1) = x(t_0) + p(t_r) \mathcal{D}_\mathrm{PM} $$

							  state = flowpm.lpt_init(initial_conds, a0=0.1)

							  # Evolve particles down to z=0
							  final_state = flowpm.nbody(state, stages, 32)

							  # Retrieve final density field
							  final_field = flowpm.cic_paint(tf.zeros_like(initial_conditions),
											 final_state[0])

							  with tf.Session() as sess:
							      sim = sess.run(final_field)

The practical challenges



  • Simulations of scientifically interesting sizes do not fit on a single GPU RAM
    e.g. $128^3$ operational, need $1024^3$ for survey volumes
    $\Longrightarrow$ We need for a distributed Machine Learning Framework

  • Most common form of distribution is data-parallelism $\Longrightarrow$ Reached Exascale on scientific deep learning applications

  • What we need is model-parallelism on HPC environments


$\Longrightarrow$ We have started investigating Mesh TensorFlow at NERSC.

Mesh TensorFlow in a few words

Redefines the TensorFlow API in terms of abstract logical tensors to operate on a
mesh : an n-dimensional array of processors, conneced by a network

  • Each tensor is distributed (split/replicated) across all processors in the mesh. To instantiate this :

    • Specify the mesh layout of computing devices with named dimensions
    • Specify names of tensor dimensions (to allow for logical dimensions)
    • Computation layout : tensor dimension $\rightarrow$ mesh-dimension
